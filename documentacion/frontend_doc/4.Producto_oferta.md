# 📦 Producto_Oferta

La carpeta **`frontend/js/producto_oferta/`** contiene toda la lógica relacionada con la visualización del catálogo de productos del sistema.

Su principal objetivo consiste en obtener los productos desde la API REST, renderizarlos dinámicamente en la interfaz y permitir al usuario navegar por el catálogo, visualizar los detalles de un producto y agregarlo al carrito de compras.

A diferencia del módulo **carrito**, esta carpeta trabaja principalmente sobre la información proveniente del Backend, transformando los datos recibidos en componentes HTML interactivos.

---

# 📚 ¿Qué responsabilidades tiene esta carpeta?

Los módulos pertenecientes a esta carpeta permiten:

- Obtener productos desde la API REST.
- Mostrar únicamente los productos activos.
- Filtrar productos por categoría.
- Construir dinámicamente las tarjetas del catálogo.
- Mostrar el detalle de un producto.
- Administrar la paginación.
- Agregar productos al carrito.
- Mostrar mensajes cuando no existen resultados.

# 📄 productos_api.js

Este módulo concentra todas las solicitudes HTTP relacionadas con los productos.

Su objetivo consiste en aislar completamente la comunicación con el Backend.

De esta forma, el resto del Frontend únicamente invoca funciones sin preocuparse por cómo se realizan las peticiones.

---

# obtenerProductos()

## Responsabilidad

Obtener el listado completo de productos disponibles.

---

## Funcionamiento

La función realiza una petición HTTP utilizando el método:

```http
GET /api/productos
```

El Backend responde con un arreglo de objetos en formato JSON.

Posteriormente la función devuelve dicho arreglo para que pueda ser utilizado por el resto de los módulos.

---

# obtenerProductoPorId(id)

## Responsabilidad

Solicitar la información completa de un producto específico.

---

## Parámetro

```text
id
```

Identificador del producto.

---

## Funcionamiento

Realiza una solicitud:

```http
GET /api/productos/:id
```

La respuesta contiene información como:

- Nombre.
- Descripción.
- Precio.
- Categoría.
- Imagen.
- Estado.

Esta información es utilizada posteriormente por la página de detalle.

---

# 📄 render_productos.js

Este módulo es responsable de construir visualmente el catálogo.

Su función principal consiste en transformar el arreglo de productos recibido desde la API en tarjetas HTML visibles para el usuario.

---

# renderizarProductos()

## Responsabilidad

Generar dinámicamente todas las tarjetas del catálogo.

---

## Funcionamiento

La función realiza el siguiente proceso:

1. Limpia el contenedor principal.
2. Recorre el arreglo recibido.
3. Crea una tarjeta para cada producto.
4. Inserta la imagen correspondiente.
5. Muestra el nombre.
6. Muestra el precio.
7. Agrega el botón **Ver Detalle**.
8. Agrega el botón **Agregar al Carrito**.

Finalmente inserta todas las tarjetas dentro del DOM.

---

## Tarjeta de Producto

Cada producto genera una estructura similar a:

```text
Imagen

Nombre

Precio

Ver detalle

Agregar al carrito
```

Toda esta estructura es creada mediante JavaScript.

---

# mostrarSinResultados()

## Responsabilidad

Mostrar un mensaje cuando no existen productos disponibles.

---

## Funcionamiento

Cuando el arreglo recibido está vacío se reemplaza el contenido del catálogo por un mensaje indicando que no existen productos para mostrar.

Esto evita que la página permanezca completamente vacía y mejora la experiencia del usuario.

---

# 📄 detalle_producto.js

Este módulo administra la página de detalle de cada producto.

---

# cargarDetalleProducto()

## Responsabilidad

Obtener toda la información de un producto específico.

---

## Funcionamiento

La función:

1. Obtiene el identificador del producto desde la URL.
2. Consulta la API REST.
3. Espera la respuesta.
4. Construye la vista del producto.
5. Inserta el contenido dentro del documento HTML.

---

## Información mostrada

La vista incluye:

- Imagen.
- Nombre.
- Descripción.
- Precio.
- Categoría.
- Botón Agregar al Carrito.

---

# agregarAlCarritoDesdeDetalle()

## Responsabilidad

Agregar directamente el producto visualizado al carrito.

---

## Funcionamiento

La función reutiliza el módulo del carrito.

Una vez agregado el producto:

- Actualiza el contador.
- Muestra un mensaje de confirmación.
- Mantiene al usuario dentro de la página de detalle.

Esto evita interrumpir la navegación.

---

# 📄 paginacion.js

Este módulo administra la navegación entre páginas del catálogo.

---

# generarPaginacion()

## Responsabilidad

Crear automáticamente los botones de navegación.

---

## Funcionamiento

A partir de la cantidad total de productos calcula:

- Cantidad de páginas.
- Página actual.
- Botones necesarios.

Luego genera dinámicamente los controles de navegación.

---

# cambiarPagina()

## Responsabilidad

Actualizar el catálogo cuando el usuario selecciona una página distinta.

---

## Funcionamiento

La función:

- Calcula el nuevo rango de productos.
- Vuelve a renderizar únicamente esos elementos.
- Actualiza la paginación resaltando la página activa.

---

# Ventajas

La paginación evita cargar una gran cantidad de productos simultáneamente.

Esto mejora:

- Rendimiento.
- Tiempo de carga.
- Experiencia de navegación.

---

# 📄 productos_main.js

Es el punto de entrada del módulo.

Su función consiste en inicializar completamente el catálogo cuando la página termina de cargarse.

---

# DOMContentLoaded

Cuando el documento está listo:

- Solicita los productos.
- Filtra la categoría correspondiente.
- Genera la primera página.
- Renderiza el catálogo.
- Configura los eventos de navegación.
- Actualiza el contador del carrito.

---

# Filtrado por Categoría

Dependiendo de la página visitada:

```text
remeras.html

↓

Categoría Remeras
```

o

```text
pantalones.html

↓

Categoría Pantalones
```

Únicamente se muestran los productos pertenecientes a dicha categoría.

---

# Comunicación con el Carrito

El módulo de productos trabaja conjuntamente con el módulo **carrito**.

Cuando el usuario presiona:

```text
Agregar al carrito
```

La información del producto es enviada directamente al módulo encargado de administrar las compras.

Esta comunicación evita duplicar código y mantiene las responsabilidades claramente separadas.

---

# Renderizado Dinámico

Todo el catálogo se genera utilizando JavaScript.

Inicialmente el HTML contiene únicamente un contenedor vacío.

```html
<div id="contenedor-productos"></div>
```

Posteriormente JavaScript crea todos los elementos mediante funciones como:

```javascript
createElement()

appendChild()

innerHTML
```

Este enfoque permite actualizar el contenido sin necesidad de recargar la página.
