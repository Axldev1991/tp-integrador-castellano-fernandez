# 🎨 Frontend

El Frontend representa la interfaz con la que interactúan los clientes del sistema de autoservicio.

Fue desarrollado utilizando tecnologías web estándar:

- HTML5
- CSS3
- JavaScript (ES6)

A diferencia del Back-Office administrativo, el Frontend **no utiliza EJS**, ya que todas las páginas son estáticas y el contenido dinámico se obtiene mediante llamadas a la API REST del Backend.

---

# 🏗️ Arquitectura del Frontend

El Frontend fue organizado de manera modular para separar la presentación, los estilos y la lógica de negocio.

```text
frontend/

assets/
│
css/
│
js/
│
pages/
│
index.html
```

Cada carpeta posee una responsabilidad específica.

---

# 📂 assets/

Contiene todos los recursos gráficos del proyecto.

Ejemplos:

- Logo
- Banner
- Iconos
- Imágenes de productos
- Favicon

Estos archivos son utilizados por las distintas páginas HTML para mejorar la experiencia visual del usuario.

---

# 📂 css/

Contiene todas las hojas de estilo del sistema.

Cada archivo se encarga de un módulo distinto.

Ejemplo:

```text
global.css

navbar.css

cards.css

productos.css

carrito.css

ticket.css

encuesta.css
```

Esta separación evita crear un único archivo CSS enorme y facilita el mantenimiento.

---

# 📂 js/

Contiene toda la lógica del Frontend.

Aquí se implementan funcionalidades como:

- Consumo de la API.
- Carrito de compras.
- Renderizado de productos.
- Cambio de tema.
- Manejo de sesiones.
- Encuestas.
- Tickets.
- Componentes reutilizables.

Todo el comportamiento dinámico del sitio se desarrolla mediante JavaScript.

---

# 📂 pages/

Contiene todas las páginas HTML del sistema.

Cada archivo representa una pantalla distinta del kiosco virtual.

```text
remeras.html

pantalones.html

detalle.html

carrito.html

ticket.html

encuesta.html
```

---

# 📄 index.html

## Responsabilidad

Es la página principal del sistema.

Representa el punto de ingreso al kiosco virtual.

Desde aquí el usuario puede iniciar la navegación hacia el catálogo de productos.

Generalmente contiene:

- Presentación del comercio.
- Logo.
- Acceso al catálogo.
- Información introductoria.

---

## Componentes

- Lista del carrito.



- Se genera el ticket.
- Se limpia el carrito.
- Se redirecciona al comprobante.

# 📚 Header y Footer dinámicos

En lugar de copiar el mismo código HTML en todas las páginas, el proyecto carga automáticamente el Header y el Footer mediante JavaScript.

Cada página únicamente posee:

```html
<header></header>

<footer></footer>
```

Posteriormente el archivo:

```text
load_components.js
```

inserta el contenido correspondiente.

Esta técnica reduce considerablemente la duplicación de código.

---

# 🌙 Cambio de Tema

Todas las páginas incorporan un selector de tema.

```html
<div id="themeToggle">
```

Permite alternar entre:

- Tema claro.
- Tema oscuro.

El estado seleccionado permanece almacenado para mantener la preferencia del usuario durante la navegación. :contentReference[oaicite:12]{index=12}

---

# 🛒 Carrito Flotante

Las páginas del catálogo muestran un botón flotante.

```html
<a class="carrito">
```

Su función consiste en permitir acceder rápidamente al carrito de compras.

También incorpora un contador que indica la cantidad de productos agregados.

Este contador se actualiza automáticamente mediante JavaScript. :contentReference[oaicite:13]{index=13}

# 📌 Buenas prácticas implementadas

Durante el desarrollo del Frontend se aplicaron diversas buenas prácticas:

- Separación entre estructura (HTML), estilos (CSS) y lógica (JavaScript).
- Uso de componentes reutilizables para Header y Footer.
- Renderizado dinámico mediante JavaScript.
- Organización modular de scripts.
- Navegación consistente en todas las páginas.
- Uso de identificadores (`id`) para manipular el DOM de manera eficiente.
- Diseño preparado para consumir una API REST.

# 📌 Resumen

| Archivo | Responsabilidad |
|----------|-----------------|
| index.html | Página principal del kiosco virtual. |
| remeras.html | Mostrar el catálogo principal de remeras. |
| pantalones.html | Mostrar el catálogo de pantalones. |
| detalle.html | Visualizar la información detallada de un producto. |
| carrito.html | Administrar los productos seleccionados antes de realizar la compra. |
| ticket.html | Mostrar el comprobante final y permitir su descarga en PDF. |
| encuesta.html | Registrar la satisfacción del cliente luego de finalizar la compra. |

El Frontend constituye la interfaz principal utilizada por los clientes del sistema. Mediante HTML, CSS y JavaScript ofrece una experiencia interactiva que permite recorrer el catálogo, consultar productos, administrar un carrito de compras, registrar ventas y completar una encuesta de satisfacción, comunicándose constantemente con la API REST desarrollada en el Backend.
