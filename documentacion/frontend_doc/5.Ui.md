# 🎨 UI

La carpeta **`frontend/js/ui/`** contiene todos los módulos encargados de administrar la interfaz gráfica del sistema.

A diferencia de otros módulos que trabajan directamente con la lógica del negocio o la comunicación con la API, esta carpeta se enfoca exclusivamente en mejorar la experiencia del usuario.

Aquí se encuentran todas las funciones relacionadas con componentes reutilizables, cambios visuales y elementos compartidos entre las distintas páginas del Frontend.

---

# 📚 ¿Qué responsabilidades tiene esta carpeta?

Los módulos de la carpeta **ui** permiten:

- Cargar automáticamente el Header y Footer.
- Cambiar entre tema claro y oscuro.
- Actualizar el contador del carrito.
- Mostrar el nombre del usuario.
- Mantener una interfaz consistente en todas las páginas.
- Evitar la duplicación de código HTML.

# 📄 load_components.js

Este archivo es responsable de insertar automáticamente los componentes compartidos por todas las páginas.

Gracias a este módulo, no es necesario copiar el mismo código HTML en cada archivo del proyecto.

---

# cargarComponentesComunes()

## Responsabilidad

Insertar dinámicamente el Header y el Footer.

---

## Funcionamiento

Cuando la página termina de cargarse:

1. Busca el elemento `<header>`.
2. Inserta el contenido correspondiente.
3. Busca el elemento `<footer>`.
4. Inserta el Footer.
5. Registra los eventos asociados a dichos componentes.

---

## Ventajas

Sin este sistema sería necesario repetir cientos de líneas de HTML en cada página.

Si el Header cambia, únicamente debe modificarse este módulo.

---

Mientras que el Footer contiene:

# 📄 theme.js

Este módulo administra el cambio entre tema claro y oscuro.

---

# cargarTema()

## Responsabilidad

Aplicar el último tema seleccionado por el usuario.

---

## Funcionamiento

Cuando la página inicia:

1. Lee la configuración almacenada en `localStorage`.
2. Determina cuál fue el último tema utilizado.
3. Aplica automáticamente las clases CSS correspondientes.

Esto evita que el usuario deba seleccionar nuevamente su preferencia cada vez que ingresa al sistema.

---

# cambiarTema()

## Responsabilidad

Alternar entre los distintos temas disponibles.

---

## Funcionamiento

Cuando el usuario presiona el botón correspondiente:

- Si el tema actual es claro, cambia al oscuro.
- Si el tema actual es oscuro, vuelve al claro.

Posteriormente guarda la preferencia utilizando:

```javascript
localStorage
```

De esta forma la selección permanece disponible durante futuras visitas.

---

# Beneficios

Esta funcionalidad mejora considerablemente la experiencia del usuario y demuestra el uso de persistencia del lado del cliente.

---

# 📄 contador_carrito.js

Este módulo mantiene sincronizado el contador visible del carrito.

---

# actualizarContadorCarrito()

## Responsabilidad

Actualizar la cantidad de productos mostrada junto al ícono del carrito.

---

## Funcionamiento

La función:

1. Obtiene el carrito almacenado.
2. Calcula la cantidad total de productos.
3. Actualiza el contenido del contador.
4. Si el carrito está vacío, muestra el valor cero.

Cada vez que un producto es agregado o eliminado, esta función es ejecutada automáticamente.

---

# ¿Por qué existe este módulo?

Sin un módulo específico, cada archivo debería actualizar manualmente el contador.

Centralizando esta responsabilidad se evita código duplicado.

Además, cualquier modificación futura solamente deberá realizarse en un único lugar.

---

# Comunicación con otros módulos

El contador trabaja conjuntamente con el módulo **carrito**.

Cuando ocurre cualquiera de las siguientes acciones:

- Agregar producto.
- Eliminar producto.
- Vaciar carrito.
- Confirmar compra.

El módulo **carrito** invoca automáticamente:

```javascript
actualizarContadorCarrito()
```

De esta manera la información mostrada al usuario siempre permanece sincronizada.

---

# Manipulación del DOM

Los módulos de UI modifican distintos elementos del documento HTML utilizando funciones como:

```javascript
document.querySelector()

document.getElementById()

classList.add()

classList.remove()

innerHTML

textContent
```

Estas operaciones permiten actualizar únicamente los elementos necesarios sin volver a cargar toda la página.

---

# Persistencia

La carpeta UI utiliza `localStorage` para conservar ciertas preferencias del usuario.

Entre ellas:

- Tema seleccionado.
- Nombre del usuario.
- Configuración de la interfaz.

Esto permite mantener una experiencia consistente durante toda la navegación.
