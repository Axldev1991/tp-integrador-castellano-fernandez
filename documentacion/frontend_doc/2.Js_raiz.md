# ⚙️ JavaScript (Archivos Raíz)

La carpeta **`frontend/js/`** contiene, además de los distintos módulos organizados en carpetas, una serie de archivos JavaScript encargados de coordinar el funcionamiento general del Frontend.

Estos archivos no pertenecen a un módulo específico porque cumplen funciones globales que son utilizadas por varias páginas del sistema.

Entre sus responsabilidades se encuentran:

- Gestionar la sesión del usuario.
- Verificar permisos de acceso.
- Inicializar la aplicación.
- Coordinar la comunicación entre los distintos módulos.

---

# 📚 ¿Qué responsabilidades tienen estos archivos?

Los archivos ubicados en la raíz de `js/` permiten:

- Inicializar el Frontend.
- Administrar la sesión del cliente.
- Validar si un usuario puede acceder a determinadas páginas.
- Redireccionar cuando corresponde.
- Compartir información entre los distintos módulos.
- Preparar la interfaz antes de que el usuario interactúe con ella.

# 📄 sesion.js

Este archivo administra toda la información relacionada con la sesión del usuario dentro del Frontend.

A diferencia del panel administrativo, el sistema cliente no utiliza sesiones del servidor.

La información se almacena utilizando `localStorage`, permitiendo mantener ciertos datos durante toda la navegación.

---

# iniciarSesion()

## Responsabilidad

Registrar el inicio de una sesión del cliente.

---

## Funcionamiento

La función recibe la información ingresada por el usuario y la almacena en el navegador utilizando `localStorage`.

Generalmente registra información como:

- Nombre.
- Correo electrónico.
- Fecha de ingreso.

Una vez almacenados los datos, el resto de las páginas puede acceder a ellos sin necesidad de volver a solicitarlos.

---

# obtenerSesion()

## Responsabilidad

Recuperar la información almacenada del usuario.

---

## Funcionamiento

Lee los datos guardados previamente en `localStorage`.

Si la sesión existe devuelve un objeto con la información del usuario.

Si no existe, devuelve un valor nulo o vacío para indicar que no hay un cliente identificado.

---

# cerrarSesion()

## Responsabilidad

Finalizar la sesión actual.

---

## Funcionamiento

La función elimina toda la información relacionada con el usuario almacenada en `localStorage`.

Entre los datos eliminados pueden encontrarse:

- Nombre del cliente.
- Información temporal de la sesión.
- Preferencias asociadas al usuario.

Una vez eliminados los datos, el sistema redirecciona nuevamente a la pantalla inicial.

---

# mostrarNombreUsuario()

## Responsabilidad

Actualizar la interfaz mostrando el nombre del usuario actualmente identificado.

---

## Funcionamiento

La función obtiene la información almacenada en la sesión y reemplaza dinámicamente el contenido del elemento HTML correspondiente.

Esto permite personalizar la experiencia del usuario durante toda la navegación.

---

# 📄 verificar_sesion.js

Este archivo se ejecuta automáticamente cuando determinadas páginas requieren que el usuario haya iniciado una sesión previamente.

Su función consiste en impedir que un usuario acceda directamente a páginas protegidas sin haber completado el flujo correspondiente.

---

# verificarSesion()

## Responsabilidad

Comprobar si existe una sesión válida.

---

## Funcionamiento

El proceso realizado es el siguiente:

1. Leer la información almacenada en `localStorage`.
2. Verificar si existe un usuario registrado.
3. Si la sesión es válida, permitir la navegación.
4. Si no existe información, redireccionar automáticamente al inicio.

---

# ¿Por qué es necesaria esta validación?

Sin esta verificación un usuario podría ingresar manualmente una dirección como:

```text
ticket.html
```

o

```text
encuesta.html
```

sin haber realizado ninguna compra.

Este archivo evita ese comportamiento, garantizando que el flujo de navegación sea correcto.

---

# Redirecciones

Cuando la validación falla, el usuario es enviado automáticamente hacia la página principal.

Esto evita errores y mantiene la consistencia del sistema.

---

# 📄 main.js

El archivo **main.js** constituye el punto de entrada general del Frontend.

Es el encargado de coordinar la inicialización de los distintos módulos utilizados por cada página.

---

# inicializarAplicacion()

## Responsabilidad

Preparar completamente la interfaz cuando el documento termina de cargarse.

---

## Funcionamiento

Cuando ocurre el evento:

```javascript
DOMContentLoaded
```

la función realiza diversas tareas de inicialización.

Entre ellas:

- Verificar la sesión.
- Cargar Header y Footer.
- Aplicar el tema almacenado.
- Actualizar el contador del carrito.
- Inicializar los módulos correspondientes a la página actual.

De esta forma todos los componentes comienzan a funcionar correctamente antes de que el usuario interactúe con ellos.

---

# registrarEventos()

## Responsabilidad

Registrar todos los eventos principales del sistema.

---

## Funcionamiento

Dependiendo de la página actual, registra eventos asociados a:

- Botones.
- Formularios.
- Navegación.
- Cambio de tema.
- Carrito.
- Productos.

Esta centralización facilita la organización del código y evita registrar eventos duplicados.

---

# Coordinación de módulos

Uno de los objetivos principales de **main.js** consiste en coordinar la comunicación entre los distintos módulos del Frontend.

El archivo no implementa directamente la lógica del negocio.

Su función consiste en llamar a los distintos módulos en el orden adecuado.

Por ejemplo:

```text
Inicializar

↓

UI

↓

Productos

↓

Carrito

↓

Sesión
```

---

# Comunicación con localStorage

Estos archivos utilizan frecuentemente `localStorage` para compartir información entre distintas páginas.

Entre los datos almacenados se encuentran:

- Usuario actual.
- Tema seleccionado.
- Carrito.
- Último ticket generado.

Esto permite que el sistema mantenga su estado durante toda la navegación sin depender constantemente del servidor.

# 📌 Buenas Prácticas Aplicadas

Durante el desarrollo de estos archivos se implementaron diversas buenas prácticas:

- Separación entre inicialización y lógica de negocio.
- Centralización del manejo de sesiones.
- Uso de `localStorage` para persistencia temporal.
- Verificación automática de permisos de acceso.
- Coordinación modular de la aplicación.
- Inicialización controlada mediante `DOMContentLoaded`.
