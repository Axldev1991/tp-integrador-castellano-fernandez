# 🛒 Carrito

La carpeta **`frontend/js/carrito/`** contiene toda la lógica relacionada con el carrito de compras del sistema.

Su objetivo es administrar los productos seleccionados por el cliente antes de confirmar la compra.

Para ello utiliza el almacenamiento local del navegador (`localStorage`), permitiendo que los productos permanezcan disponibles incluso si el usuario cambia de página o recarga el sitio. :contentReference[oaicite:0]{index=0}

---

# 📚 ¿Qué responsabilidades tiene esta carpeta?

Los módulos del carrito permiten:

- Agregar productos.
- Eliminar productos.
- Modificar cantidades.
- Calcular subtotales.
- Calcular el total de la compra.
- Persistir el carrito mediante `localStorage`.
- Mostrar el carrito dinámicamente.
- Registrar la compra en el Backend.

# 📄 carrito_data.js

Este módulo administra toda la información almacenada dentro del carrito.

Aquí se realizan todas las operaciones sobre el arreglo de productos.

El carrito se inicializa leyendo la información almacenada previamente en `localStorage`. Si no existe información, se crea un arreglo vacío. :contentReference[oaicite:1]{index=1}

---

# guardarCarrito()

## Responsabilidad

Guardar el estado actual del carrito.

---

## Funcionamiento

Convierte el arreglo de productos en formato JSON utilizando `JSON.stringify()` y posteriormente lo almacena en `localStorage`.

Esto garantiza que la información permanezca disponible aunque el usuario abandone la página. :contentReference[oaicite:2]{index=2}

---

# agregarAlCarrito(producto)

## Responsabilidad

Agregar un nuevo producto al carrito.

---

## Parámetros

```text
producto
```

Objeto que contiene toda la información del producto seleccionado.

---

## Funcionamiento

1. Busca si el producto ya existe en el carrito.
2. Si existe, incrementa la cantidad.
3. Si no existe, crea un nuevo registro con cantidad inicial igual a uno.
4. Guarda el carrito actualizado.
5. Muestra un mensaje indicando que el producto fue agregado.
6. Actualiza automáticamente el contador del carrito. :contentReference[oaicite:3]{index=3}

---

# quitarUnaUnidad(id)

## Responsabilidad

Disminuir la cantidad de un producto.

---

## Parámetro

```text
id
```

Identificador del producto.

---

## Funcionamiento

Si el producto posee más de una unidad, reduce su cantidad.

Si únicamente queda una unidad, elimina completamente el producto del carrito.

Finalmente:

- Guarda los cambios.
- Renderiza nuevamente el carrito.
- Actualiza el contador. :contentReference[oaicite:4]{index=4}

---

# eliminarProducto(id)

## Responsabilidad

Eliminar completamente un producto del carrito.

---

## Funcionamiento

Filtra el arreglo eliminando el producto cuyo identificador coincide con el recibido.

Luego:

- Guarda el carrito.
- Actualiza la interfaz.
- Notifica el cambio al contador flotante. :contentReference[oaicite:5]{index=5}

# 📄 carrito_render.js

Este módulo se encarga exclusivamente de representar visualmente el contenido del carrito.

Toda modificación realizada sobre los datos termina llamando nuevamente a esta función para actualizar la interfaz. :contentReference[oaicite:6]{index=6}

---

# renderizarCarrito()

## Responsabilidad

Construir dinámicamente toda la vista del carrito.

---

## Funcionamiento

La función realiza el siguiente proceso:

1. Obtiene el contenedor HTML del carrito.
2. Verifica si existen productos.
3. Si el carrito está vacío muestra un mensaje junto con un enlace para continuar comprando.
4. Si existen productos, genera dinámicamente una tarjeta para cada uno.
5. Calcula el subtotal de cada producto.
6. Calcula el total general.
7. Actualiza el contenido HTML del carrito.
8. Registra todos los eventos de los botones (+, -, eliminar). 

---

## Productos Vacíos

Cuando no existen productos el usuario observa un mensaje similar a:

```text
🛒 Tu carrito está vacío

Seguir comprando
```

En este caso el total mostrado es:

```text
$0
```

---

## Cálculo del Total

Durante el recorrido del carrito la función calcula automáticamente:

```text
subtotal = precio × cantidad
```

Posteriormente suma todos los subtotales para obtener el importe final de la compra. :contentReference[oaicite:8]{index=8}

---

## Eventos registrados

Cada producto incorpora tres botones.

### Botón +

Incrementa la cantidad del producto.

---

### Botón -

Reduce la cantidad.

Si únicamente queda una unidad, elimina el producto.

---

### Botón ✖

Elimina completamente el producto seleccionado.

# 📄 carrito_actions.js

Este módulo reúne todas las acciones de mayor nivel relacionadas con la compra.

Mientras que `carrito_data.js` administra los datos y `carrito_render.js` dibuja la interfaz, este archivo coordina las operaciones principales del proceso de compra. :contentReference[oaicite:10]{index=10}

---

# vaciarCarrito()

## Responsabilidad

Eliminar todos los productos del carrito.

---

## Funcionamiento

1. Vacía completamente el arreglo.
2. Guarda el nuevo estado.
3. Renderiza nuevamente el carrito para reflejar los cambios. :contentReference[oaicite:11]{index=11}

---

# finalizarCompra()

## Responsabilidad

Iniciar el proceso de compra.

---

## Funcionamiento

Antes de continuar verifica si existen productos.

Si el carrito está vacío muestra un mensaje de advertencia.

Si contiene productos solicita confirmación al usuario.

Únicamente después de aceptar se ejecuta la compra definitiva. :contentReference[oaicite:12]{index=12}

---

# confirmarCompra()

## Responsabilidad

Registrar la venta.

Es la función más importante del módulo.

---

## Funcionamiento

1. Obtiene el nombre del cliente desde `localStorage`.
2. Genera la fecha actual.
3. Construye el objeto `venta`.
4. Calcula subtotales y total general.
5. Envía la información al Backend mediante una petición `POST`.
6. Si la operación es exitosa:
   - Guarda el ticket en `localStorage`.
   - Vacía el carrito.
   - Redirecciona a `ticket.html`.
7. Si ocurre un error muestra un mensaje al usuario. :contentReference[oaicite:13]{index=13}

---

## Comunicación con el Backend

La venta se registra utilizando:

```http
POST /api/ventas
```

El cuerpo de la petición contiene:

- Cliente.
- Fecha.
- Productos.
- Cantidades.
- Total.

Esta información es procesada posteriormente por el Backend y almacenada en la base de datos. :contentReference[oaicite:14]{index=14}

# DOMContentLoaded

Cuando el documento termina de cargarse la función:

- Obtiene el nombre del usuario.
- Muestra el saludo personalizado.
- Renderiza el carrito.
- Registra el evento del botón **Finalizar Compra**.
- Registra el evento para cerrar la sesión del usuario. :contentReference[oaicite:16]{index=16}
