# 🛡️ Middlewares

Los **Middlewares** son funciones que se ejecutan **entre la llegada de una petición HTTP y el controlador**.

Su objetivo es realizar tareas previas antes de que el controlador procese la solicitud.

Algunos ejemplos de tareas que realizan son:

- Validar información enviada por el usuario.
- Verificar que un usuario haya iniciado sesión.
- Procesar archivos.
- Registrar información.
- Modificar la petición (`req`).
- Interrumpir una petición si existe algún error.

---

# 📚 ¿Cómo funciona un Middleware?

Cuando Express recibe una petición, puede ejecutar una cadena de funciones antes de llegar al controlador.

Por ejemplo:

```javascript
router.post(
    "/productos/nuevo",
    isAuthenticated,
    upload.single("imagen"),
    validateProduct,
    postNuevoProducto
);
```

El flujo es el siguiente:

```text
Cliente

↓

isAuthenticated

↓

upload.single()

↓

validateProduct()

↓

postNuevoProducto()
```

Cada middleware decide si la petición puede continuar o no.

Si todo está correcto ejecuta:

```javascript
next();
```

Si detecta algún problema puede responder inmediatamente al cliente sin ejecutar el controlador.

---

# ¿Qué es next()?

`next()` es una función proporcionada por Express.

Su única responsabilidad consiste en indicar que la petición puede continuar hacia el siguiente middleware o controlador.

Ejemplo:

```javascript
export const ejemplo = (req,res,next)=>{

    console.log("Middleware ejecutado");

    next();

}
```

Si `next()` nunca se ejecuta, la petición queda detenida y Express nunca llegará al controlador.

# 🔐 auth.js

## Responsabilidad

Controlar el acceso al panel administrativo.

Este archivo evita que usuarios no autenticados puedan acceder a páginas privadas.

Contiene dos middlewares.

```text
isAuthenticated()

isAdmin()
```

---

# isAuthenticated()

## Responsabilidad

Verificar que exista una sesión iniciada.

---

## Funcionamiento

El middleware consulta la sesión creada por Express Session.

```javascript
req.session.usuario
```

Si existe un usuario almacenado:

```javascript
return next();
```

La petición continúa normalmente.

---

Si no existe:

```javascript
res.redirect("/admin/login");
```

El usuario es enviado automáticamente al Login.

---

## Flujo

```text
Cliente

↓

¿Existe req.session.usuario?

↓

Sí
↓

Dashboard

No
↓

Login
```

---

## ¿Por qué utilizar sesiones?

HTTP es un protocolo sin memoria.

Cada petición es independiente.

Si un administrador inicia sesión y luego abre el Dashboard, el servidor no recuerda automáticamente quién es.

Las sesiones permiten almacenar información del usuario entre múltiples peticiones.

Gracias a ello el administrador solamente inicia sesión una vez.

---

## Ejemplo de uso

```javascript
router.get(
"/dashboard",
isAuthenticated,
getDashboard
);
```

Antes de ejecutar:

```
getDashboard()
```

Express verifica que el usuario haya iniciado sesión.

---

# isAdmin()

## Responsabilidad

Verificar permisos administrativos.

Actualmente realiza la misma comprobación que `isAuthenticated()`.

Sin embargo fue creada pensando en futuras ampliaciones del sistema.

Por ejemplo:

```text
Administrador

Empleado

Supervisor
```

Cada rol podría acceder únicamente a determinadas funcionalidades.

---

Actualmente el flujo es:

```text
¿Existe sesión?

↓

Sí

↓

next()

↓

No

↓

redirect(Login)
```

# 📤 upload.js

## Responsabilidad

Procesar archivos enviados desde formularios HTML.

En este proyecto se utiliza para subir imágenes de productos y archivos de encuestas.

---

# ¿Qué es Multer?

Multer es un middleware especializado en recibir archivos binarios.

Cuando un usuario selecciona una imagen:

```text
imagen.png
```

El navegador envía un formulario especial denominado:

```
multipart/form-data
```

Express no puede interpretar este tipo de datos por sí solo.

Multer se encarga de hacerlo.

---

# Funcionamiento

El flujo completo es:

```text
Formulario

↓

Imagen

↓

Multer

↓

public/uploads/

↓

req.file

↓

Controller
```

---

# diskStorage()

El proyecto utiliza:

```javascript
multer.diskStorage()
```

Esto significa que los archivos se almacenan directamente en el disco.

No se guardan dentro de la base de datos.

---

# destination()

```javascript
destination(...)
```

Define la carpeta donde se almacenarán los archivos.

En este proyecto:

```text
public/uploads/
```

Todas las imágenes terminan dentro de ese directorio.

---

# filename()

Cada archivo recibe un nombre único.

Por ejemplo:

```
1719072387123-84723812.png
```

El nombre se genera utilizando:

- Fecha actual
- Número aleatorio
- Extensión original

Esto evita problemas cuando dos usuarios suben archivos con el mismo nombre.

---

# upload.single()

Es el middleware utilizado en la mayoría del proyecto.

Ejemplo:

```javascript
upload.single("imagen")
```

Le indica a Multer que debe esperar únicamente un archivo cuyo campo del formulario se llame:

```
imagen
```

---

Cuando termina el proceso agrega automáticamente:

```javascript
req.file
```

Ejemplo:

```javascript
req.file.filename

req.file.path

req.file.destination

req.file.originalname
```

El controlador utiliza esta información para guardar la ruta dentro de la base de datos.

---

# 📋 validators.js

## Responsabilidad

Validar toda la información enviada por el cliente.

Gracias a estas validaciones se evita guardar datos incorrectos en la base de datos.

---

# validateEmail()

## Responsabilidad

Comprobar que un correo electrónico tenga un formato válido.

Utiliza una expresión regular.

Ejemplo válido:

```
usuario@gmail.com
```

Ejemplo inválido:

```
usuariogmail.com
```

---

Devuelve:

```javascript
true
```

o

```javascript
false
```

---

# validatePositivePrice()

## Responsabilidad

Comprobar que un precio sea un número positivo.

Ejemplos válidos

```
100

15.5

9999
```

Ejemplos inválidos

```
0

-10

abc
```

---

Devuelve:

```javascript
true
```

cuando el precio es válido.

---

# validateRegister()

## Responsabilidad

Validar el formulario de registro de administradores.

---

## Campos validados

### Nombre

Debe existir.

Debe contener al menos tres caracteres.

---

### Correo

Debe tener un formato válido.

---

### Contraseña

Debe tener seis caracteres como mínimo.

---

### Confirmación

Debe coincidir exactamente con la contraseña.

---

## Si existe algún error

El middleware no ejecuta el controlador.

Renderiza nuevamente:

```
register.ejs
```

mostrando todos los errores encontrados.

---

## Si todo es correcto

Ejecuta:

```javascript
next();
```

---

# validateProduct()

## Responsabilidad

Validar el formulario de productos.

---

Comprueba:

### Nombre

Debe contener al menos dos caracteres.

---

### Descripción

Debe contener cinco caracteres como mínimo.

---

### Precio

Debe ser un número positivo.

---

### Categoría

Actualmente únicamente acepta:

```
Remeras

Pantalones
```

Si llega cualquier otro valor:

```
Camperas
```

la petición es rechazada.

---

## Error

Si existe algún problema vuelve a mostrar:

```
formulario-producto.ejs
```

con toda la información ingresada previamente para que el usuario no deba completar nuevamente el formulario.

---

# validateLogin()

## Responsabilidad

Validar el formulario de inicio de sesión.

---

Comprueba únicamente:

- correo válido
- contraseña presente

No verifica si el usuario existe.

Esa responsabilidad pertenece al controlador.

---

# validateID()

## Responsabilidad

Verificar parámetros enviados por la URL.

Ejemplo:

```
/productos/15
```

El middleware obtiene:

```javascript
req.params.id
```

Luego realiza varias comprobaciones.

---

Debe ser:

- número
- entero
- mayor que cero

---

Ejemplos válidos

```
1

25

100
```

---

Ejemplos inválidos

```
abc

0

-10

15.8
```

---

Si el ID es correcto se almacena en:

```javascript
req.id
```

De esta manera los controladores ya reciben un entero listo para utilizar.

---

Si existe un error responde inmediatamente:

```json
{
    "error":"El id debe ser un número entero positivo"
}
```

---

# validateVenta()

Es el middleware más complejo del proyecto.

---

## Responsabilidad

Validar completamente una venta antes de almacenarla.

---

## Cliente

Debe existir.

Debe contener al menos dos caracteres.

---

## Total

Debe ser un número positivo.

---

## Items

Debe existir un arreglo.

Ejemplo:

```json
[
    {
        "id":1,
        "cantidad":2,
        "precio_unitario":5000
    }
]
```

---

## Cada producto debe contener

- id válido
- cantidad positiva
- precio positivo

---

Si alguno de los productos es incorrecto se genera un error indicando exactamente cuál falló.

Ejemplo:

```text
Item 3:
cantidad debe ser un número entero positivo
```

---

Si existen errores responde:

```json
{
    "status":"error",
    "errors":[
        "...",
        "...",
        "..."
    ]
}
```

No se registra ninguna venta.

---

Si toda la información es correcta:

```javascript
next();
```

El controlador puede almacenar la venta en la base de datos.

---

# 🔄 Flujo Completo de los Middlewares

```text
Cliente

↓

Express

↓

Router

↓

Middleware 1

↓

Middleware 2

↓

Middleware 3

↓

Controller

↓

Model

↓

Base de Datos

↓

Respuesta
```

Cada middleware posee una única responsabilidad específica.

Esta separación permite reutilizar código, mejorar la organización del proyecto y mantener una arquitectura limpia y fácil de mantener.

---

# 📌 Resumen

| Middleware | Responsabilidad |
|------------|-----------------|
| isAuthenticated() | Verificar que exista una sesión iniciada antes de permitir el acceso a rutas protegidas. |
| isAdmin() | Preparar el sistema para futuras validaciones de roles administrativos. |
| upload.single() | Procesar y almacenar archivos enviados desde formularios HTML utilizando Multer. |
| validateEmail() | Verificar el formato de una dirección de correo electrónico. |
| validatePositivePrice() | Comprobar que un valor numérico sea mayor que cero. |
| validateRegister() | Validar todos los datos necesarios para registrar un nuevo administrador. |
| validateProduct() | Validar la información de un producto antes de crearlo o editarlo. |
| validateLogin() | Validar los datos mínimos requeridos para iniciar sesión. |
| validateID() | Garantizar que el identificador recibido por la URL sea un entero positivo. |
| validateVenta() | Validar la estructura completa de una venta y de cada uno de sus productos antes de almacenarla. |
