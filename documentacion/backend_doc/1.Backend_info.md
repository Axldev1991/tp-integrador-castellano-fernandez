# 🛒 Tienda de Autoservicio - TP Integrador

Trabajo Práctico Integrador correspondiente a la materia **Programación III** de la **Universidad Tecnológica Nacional (UTN)**.

El proyecto implementa un sistema completo de autoservicio para una tienda de indumentaria urbana compuesto por dos grandes módulos:

- Un **Frontend** que utilizan los clientes para visualizar productos, realizar compras y completar encuestas.
- Un **Backend** desarrollado con **Node.js** y **Express**, encargado de administrar productos, ventas, usuarios, autenticación, auditoría y comunicación con la base de datos.

El sistema sigue una arquitectura **MVC (Model - View - Controller)** y utiliza una **API REST** para comunicar el frontend con el servidor.

# 📚 ¿Qué es Node.js?

Node.js es un entorno de ejecución de JavaScript que permite ejecutar código fuera del navegador.

Gracias a Node.js es posible crear servidores web utilizando JavaScript.

En este proyecto Node.js se encarga de:

- Ejecutar el servidor.
- Escuchar peticiones HTTP.
- Comunicarse con la base de datos.
- Leer archivos.
- Procesar imágenes.
- Manejar sesiones.
- Ejecutar la lógica de negocio.

Sin Node.js el proyecto solamente sería una página HTML sin posibilidad de guardar información.

---

# 📚 ¿Qué es Express?

Express es un framework construido sobre Node.js.

Su objetivo es facilitar la creación de servidores HTTP.

En lugar de escribir cientos de líneas para interpretar cada petición, Express permite hacerlo de forma muy sencilla.

Por ejemplo:

```javascript
app.get("/productos", controlador);
```

Con una sola línea Express entiende:

- Qué URL escuchar.
- Qué método HTTP aceptar.
- Qué función ejecutar.

Gracias a Express el proyecto puede organizar todas las rutas de forma modular.

---

# 📚 ¿Qué es una API REST?

Una API REST es una interfaz que permite que dos aplicaciones se comuniquen utilizando HTTP.

En este proyecto:

Frontend
↓

API REST
↓

Backend
↓

Base de Datos

Cuando el frontend necesita los productos, realiza una petición:

```
GET /api/productos
```

El servidor consulta la base de datos y devuelve un JSON.

Ejemplo:

```json
{
    "payload":[
        {
            "id":1,
            "name":"Remera Classic",
            "price":12000
        }
    ]
}
```

El frontend nunca consulta directamente la base de datos.

Siempre se comunica mediante la API.

# 📚 ¿Qué es MVC?

MVC significa:

Model
View
Controller

Es un patrón de arquitectura utilizado para separar responsabilidades.

## Model

El Model es el encargado de comunicarse con la base de datos.

Ejemplo:

```javascript
SELECT * FROM productos;
```

Toda consulta SQL vive dentro de los modelos.

---

## View

Las Views representan la interfaz gráfica.

En este proyecto se utilizan vistas EJS.

Ejemplos:

- dashboard.ejs
- login.ejs
- register.ejs
- historial-ventas.ejs

Una vista solamente muestra información.

Nunca realiza consultas SQL.

---

## Controller

Los Controllers contienen la lógica del programa.

Son el intermediario entre las rutas y los modelos.

Ejemplo:

Cliente

↓

Controller

↓

Model

↓

Base de Datos

↓

Controller

↓

Cliente

Un controlador puede:

- Validar datos.
- Consultar modelos.
- Enviar respuestas JSON.
- Renderizar vistas.
- Manejar errores.

---

# 📚 ¿Qué es un Middleware?

Un Middleware es una función que se ejecuta antes del controlador.

Su función es interceptar una petición para realizar alguna tarea.

Ejemplo:

Cliente

↓

Middleware

↓

Controller

Algunas tareas comunes son:

- Validar datos.
- Verificar autenticación.
- Subir imágenes.
- Comprobar permisos.
- Registrar información.

Ejemplo:

```javascript
router.post(
    "/productos",
    validateProduct,
    postNuevoProducto
);
```

En este caso:

1. Se ejecuta `validateProduct`.
2. Si los datos son correctos continúa.
3. Si hay errores nunca llega al controlador.

---

# 📚 ¿Qué es una Ruta?

Las rutas indican qué función ejecutar dependiendo de la URL solicitada.

Ejemplo:

```javascript
router.get("/productos", getProductos);
```

Significa:

Cuando alguien visite

```
GET /productos
```

Express ejecutará

```javascript
getProductos()
```

---

# 📚 ¿Qué es un Controller?

Un Controller contiene toda la lógica del negocio.

No realiza consultas SQL directamente.

Su trabajo consiste en:

- Leer datos enviados por el cliente.
- Validarlos.
- Llamar al modelo correspondiente.
- Procesar la información.
- Devolver una respuesta.

---

# 📚 ¿Qué es un Model?

Un Model encapsula todas las consultas SQL.

Su objetivo es aislar completamente la base de datos del resto del sistema.

En lugar de escribir consultas SQL dentro de los controladores, éstos llaman a funciones del modelo.

Ejemplo:

```javascript
const productos =
await productModels.getProductosActivos();
```

Internamente esa función ejecuta un:

```sql
SELECT * FROM productos
```

Esto permite cambiar la base de datos sin modificar los controladores.

---

# 📚 ¿Qué es EJS?

EJS (Embedded JavaScript) es un motor de plantillas.

Permite mezclar HTML con JavaScript.

Ejemplo:

```ejs
<h1><%= usuario.nombre %></h1>
```

Cuando el servidor renderiza la página reemplaza automáticamente esa variable por el valor correspondiente.

Gracias a EJS el panel administrativo puede mostrar información dinámica sin escribir HTML manualmente.

# 📂 Flujo Completo de una Petición

Supongamos que un cliente desea visualizar los productos.

1. El navegador solicita:

```
GET /api/productos
```

2. Express recibe la petición.

3. La ruta correspondiente identifica el endpoint.

4. Se ejecutan los middlewares necesarios.

5. El controlador procesa la solicitud.

6. El controlador llama al modelo.

7. El modelo consulta MySQL.

8. MySQL devuelve los resultados.

9. El modelo retorna los datos.

10. El controlador genera un JSON.

11. Express responde al navegador.

Todo este proceso ocurre en apenas unos pocos milisegundos.
