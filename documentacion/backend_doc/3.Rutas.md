# 🛣️ Rutas (Routes)

Las **Routes** son el primer punto de contacto entre el cliente y el servidor.

Su responsabilidad es indicar **qué controlador debe ejecutarse** cuando un usuario realiza una petición HTTP.

Las rutas **no contienen lógica de negocio**, ni realizan consultas SQL.

Su única responsabilidad es recibir la petición y derivarla al controlador correspondiente.

---

# 📚 ¿Qué es un Router de Express?

Express permite dividir las rutas del proyecto en múltiples archivos mediante el objeto `Router`.

En lugar de escribir todas las rutas dentro de `index.js`, cada módulo tiene su propio archivo.

Por ejemplo:

```javascript
const router = express.Router();
```

Este objeto funciona como un mini servidor que posteriormente es conectado al servidor principal mediante:

```javascript
app.use("/admin", adminRoutes);
```

De esta manera todas las rutas administrativas quedan agrupadas.

---

# 🔄 Flujo de una Petición

Cada vez que el navegador realiza una solicitud ocurre el siguiente proceso:

```text
Cliente

↓

Express

↓

Router

↓

Middleware(s)

↓

Controller

↓

Model

↓

MySQL

↓

Controller

↓

Respuesta HTTP

↓

Cliente
```

El Router únicamente decide qué camino debe seguir la petición.

## GET /admin/login

```javascript
router.get("/login", getLogin);
```

### Responsabilidad

Mostrar la pantalla de inicio de sesión.

---

### Método HTTP

GET

---

### Recibe

Nada.

---

### Ejecuta

```javascript
getLogin()
```

---

### Devuelve

La vista

```
login.ejs
```

---

## POST /admin/login

```javascript
router.post(
"/login",
validateLogin,
postLogin
);
```

### Responsabilidad

Procesar el inicio de sesión.

---

### Flujo

```text
Formulario

↓

validateLogin

↓

postLogin

↓

Dashboard
```

---

### Middleware

#### validateLogin

Comprueba:

- Email válido.
- Contraseña ingresada.

Si existe algún error, el controlador nunca se ejecuta.

---

### Controller

```javascript
postLogin()
```

Verifica las credenciales.

Si son correctas:

- crea la sesión
- registra auditoría
- redirecciona al Dashboard

Si son incorrectas:

muestra nuevamente el formulario.

# Rutas Protegidas

Todas requieren sesión iniciada.

El middleware siempre es:

```javascript
isAuthenticated
```

---

## ¿Qué hace isAuthenticated?

Comprueba:

```javascript
req.session.usuario
```

Si existe:

continúa.

Si no existe:

redirecciona a

```
/admin/login
```

Esto evita que cualquier persona ingrese directamente al Dashboard escribiendo la URL.

```text

↓

↓

↓

↓

↓

```

# 📌 Resumen General

Las rutas son el puente entre el cliente y la lógica del sistema.

Su única responsabilidad consiste en:

- recibir peticiones HTTP;
- ejecutar los middlewares necesarios;
- invocar el controlador correspondiente.

Gracias a esta separación, el proyecto mantiene una arquitectura organizada, escalable y fácil de mantener, ya que las rutas únicamente definen el flujo de navegación mientras que toda la lógica queda delegada a los controladores y modelos.
