# 🖥️ Vistas (Views)

Las **Views** representan la capa visual del sistema.

Son las páginas que el usuario observa e interactúa dentro del navegador.

En este proyecto las vistas fueron desarrolladas utilizando **EJS (Embedded JavaScript Templates)**, un motor de plantillas que permite combinar HTML con JavaScript para generar contenido dinámico desde el servidor.

Las vistas únicamente presentan información; no contienen lógica de negocio ni realizan consultas a la base de datos.

---

# 📚 ¿Qué es EJS?

EJS (Embedded JavaScript) es un motor de vistas para Node.js.

Permite escribir HTML tradicional e insertar variables provenientes del servidor.

Por ejemplo:

```ejs
<h1><%= usuario.nombre %></h1>
```

Cuando Express renderiza esta vista, reemplaza automáticamente:

```ejs
<%= usuario.nombre %>
```

por el valor recibido desde el controlador.

Resultado final:

```html
<h1>Saulo Fernández</h1>
```

---

# ¿Por qué utilizar EJS?

Sin un motor de vistas sería necesario construir todo el HTML manualmente desde JavaScript, lo que volvería el proyecto mucho más complejo.

EJS permite:

- Reutilizar componentes.
- Mostrar información dinámica.
- Crear bucles.
- Mostrar condiciones.
- Reducir código repetido.

---

# Sintaxis básica de EJS

## Mostrar una variable

```ejs
<%= producto.nombre %>
```

---

## Ejecutar JavaScript

```ejs
<%
const total = precio * cantidad;
%>
```

---

## Condicional

```ejs
<% if(producto.activo){ %>

<p>Disponible</p>

<% } %>
```

---

## Recorrer un arreglo

```ejs
<% productos.forEach(producto => { %>

<p><%= producto.nombre %></p>

<% }) %>
```

# 📂 partials/

Las **partials** son pequeños fragmentos de HTML reutilizables.

Su objetivo es evitar duplicar código.

---

## ¿Por qué existen?

Supongamos que todas las páginas poseen el mismo encabezado.

Sin partials habría que copiar:

```html
<header>

...

</header>
```

en absolutamente todos los archivos.

Si luego se quisiera modificar el logo, habría que editar todas las páginas.

Con las partials solamente existe un único archivo.

---

# include()

Las partials se insertan utilizando:

```ejs
<%- include("partials/header") %>
```

Cuando Express renderiza la página reemplaza esa línea por el contenido completo del archivo.

# 📚 Render()

Cuando un controlador necesita mostrar una vista utiliza:

```javascript
res.render(
"dashboard",
{
productos,
ventas,
estadisticas
}
);
```

Express realiza automáticamente el siguiente proceso:

```text
Controller

↓

dashboard.ejs

↓

Inserta Variables

↓

Genera HTML

↓

Envía HTML

↓

Navegador
```

---

# Diferencia entre render() y JSON

## render()

Se utiliza para mostrar páginas.

Ejemplo:

```javascript
res.render("dashboard")
```

Respuesta:

```html
<html>

...

</html>
```

---

## json()

Se utiliza para responder una API.

```javascript
res.json({

status:"success"

})
```

Respuesta:

```json
{
"status":"success"
}
```

# 🔗 Integración entre Controllers y Views

Las vistas dependen completamente de los controladores para obtener información.

El flujo de trabajo es el siguiente:

```text
Usuario

↓

Solicita una página

↓

Express Router

↓

Controller

↓

Model

↓

Base de Datos

↓

Controller

↓

render()

↓

Vista EJS

↓

HTML generado

↓

Navegador
```

Gracias a esta arquitectura, las vistas permanecen simples y enfocadas únicamente en la presentación de la información, mientras que toda la lógica de negocio y acceso a datos permanece encapsulada en los Controllers y Models.
